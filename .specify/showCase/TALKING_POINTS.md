## THE 2-MINUTE VERSION (For corridor or elevator conversations)

> "You know how we always say we want to go faster without sacrificing quality? I've found a way.
>
> It's called Spec-Driven Development. Instead of handing a vague requirement to a developer and watching two weeks of back-and-forth happen, you run it through a structured AI framework that writes the specification for you, asks the clarifying questions upfront, designs the architecture, breaks it into tasks, and then generates all the code and tests.
>
> I used it on a personal project — Angular dashboard with weather, time, and quotes. Four days. 1,923 lines of code. 44 tests, all green. Zero manual coding. The code quality is better than what I would have written under pressure on a real project — it has accessibility attributes, error boundaries, offline fallback, reduced-motion support.
>
> For our organization: if we apply this to client delivery, the math is straightforward. A project that traditionally costs $48,000 costs $1,600 with this method. The time drops from 8–12 weeks to 4 days. Multiply that across 10 projects a year and we're talking $460,000 in savings annually, from one practice.
>
> I'm not proposing we replace developers. I'm proposing we elevate them — from writing boilerplate to writing specifications that direct AI. It's the same shift that happened when engineers moved from the drafting table to CAD software. The output got better. The speed went up. The engineers became more valuable.
>
> Can I get 30 minutes on your calendar to show you the live app and walk through the methodology?"

## KEY MESSAGE PILLARS

### Pillar 1: PROOF, NOT THEORY
**Say:** "I'm not describing a future possibility. I'm showing you something that works right now, today, on my laptop. You can clone the repository, run npm install, and see it live in 30 seconds. The code, the tests, the specification — all real, all verifiable."

**Use when:** Someone is skeptical or says "AI-generated code sounds interesting but..."

**Supporting facts:**
- git commit c3fd8d3 dated February 20, 2026
- 44/44 tests passing (run `npm test` to verify live)
- Constitution compliance table: all 6 gates green in plan.md
- 65/65 tasks marked complete in tasks.md

### Pillar 2: QUALITY IS ENFORCED, NOT HOPED FOR
**Say:** "On a normal project, accessibility is the thing we skip under deadline pressure. Error handling gets cut in the final sprint. Documentation is always 'we'll do it after.' With Spec-Driven Development, the constitution encodes those requirements as non-negotiable gates. The AI doesn't have a deadline. It doesn't skip the aria-label to save 10 minutes."

**Use when:** Leadership raises questions about code quality or technical debt.

**Supporting facts:**
- WCAG AA accessibility: role="timer", aria-labels, :focus-visible
- prefers-reduced-motion respected in CSS
- try/catch on all 3 localStorage keys
- Task T065: explicit code audit for dead code and unused imports
- Task T064: explicit accessibility verification

## THINGS TO AVOID SAYING

❌ **Don't say:** "The AI writes all the code, so we need fewer developers."
✅ **Say instead:** "The AI handles the mechanical coding, freeing developers to focus on requirements and architecture — higher value work."

❌ **Don't say:** "This is perfect for everything."
✅ **Say instead:** "This is the right tool for greenfield features, new services, and clear requirements. Legacy modernisation and ambiguous research work still need the traditional approach."

❌ **Don't say:** "We don't need to review the AI-generated code."
✅ **Say instead:** "The AI generates; the engineer reviews, validates, and owns. The review is faster because the spec makes the intent crystal clear."

❌ **Don't say:** "This will replace junior developers."
✅ **Say instead:** "Junior developers can now contribute meaningfully from day one by running specifications, rather than spending months reading legacy code before they can contribute."

❌ **Don't say:** "It's basically just ChatGPT."
✅ **Say instead:** "Claude Code is an AI agent with access to your file system that executes a structured, multi-step methodology — not a chatbot. It reads files, writes code, runs tests, and validates output in a loop."
