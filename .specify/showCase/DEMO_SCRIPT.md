>

>
> In the next ten minutes, I want to show you how that's possible, and why it matters for our organization."

## SEGMENT 2 — Project Context (1 minute)

*[Stay on the running dashboard]*

> "Let me give you the context. The requirement was straightforward: 'Build a personal dashboard that shows weather, time, and a daily quote. Make it responsive with smooth animations.'
>
> Traditional timeline for this — a 3-person squad, 8 to 12 weeks, around $48,000 in delivery cost.
>
> With Spec-Driven Development: one person, 4 days, roughly $1,600 in effort cost. Same quality. Actually better quality — because the AI enforces standards that humans under deadline pressure often skip.
>
> The method is called Spec-Driven Development. It's a 7-step framework where you write a precise specification, and the AI handles the implementation. Let me show you the artifacts first, then the code."

---

## SEGMENT 3 — The Specification Artifacts (2 minutes)

*[Switch to VS Code, open `specs/001-personal-dashboard/spec.md`]*

> "This is the specification. It was generated by AI from my original one-sentence requirement. Look at what it contains."

*[Scroll slowly through the spec]*

> "Four user stories — Time Display, Weather Widget, Quote Widget, Responsive Layout — each prioritised P1 through P4. Each story has acceptance scenarios written in Gherkin-style 'Given / When / Then' format.
>
> For example — User Story 2: 'Given location permission is denied, When the dashboard loads, Then the user is prompted to enter a city name manually.' That's not a note in a Confluence doc. That is a precise test criterion the AI used to write the code AND the automated test for it."

*[Open `specs/001-personal-dashboard/plan.md`, scroll to Constitution Check table]*

> "Now look at the plan. Before a single line of code was generated, the AI ran a compliance check against our project constitution — five governance principles. Every gate: pass. 'Are we introducing architectural patterns before validation? Pass. Is the implementation completable in one session? Pass.'
>
> This is your audit trail. Any stakeholder can look at this table and understand every architectural decision before they read any code."

*[Open `specs/001-personal-dashboard/tasks.md`, scroll through phases]*

> "And this is the task list. 65 tasks. 7 phases. Every task has a file path, a priority, a story label, and a completion marker. Every single one of these boxes is ticked. Zero deferred. Zero skipped."

## SEGMENT 4 — The Live Application (2 minutes)

*[Switch back to browser — http://localhost:4200]*

> "Back to the app. Let me show you each widget."

*[Point to Time Widget]*
> "Time Widget — Angular Signals updating every second. Uses setInterval with proper cleanup on component destruction. Updates automatically at midnight — no page refresh needed."

*[Point to Weather Widget]*
> "Weather Widget — this is the most complex piece. It has a 6-state machine: idle, locating, loading, loaded, error, and city_input. It tried geolocation first. If you deny permission, it drops into city input mode. It caches data for 60 minutes in localStorage — so return visits show data instantly."

*[Resize the browser window from full-width to narrow]*
> "Watch the layout as I resize — 3 columns on desktop, 2 on tablet, 1 on mobile. No CSS framework. Pure CSS Grid with auto-fit minmax(280px, 1fr). The widgets rearrange automatically."

*[Resize back to full width]*
> "Every widget animates in on load — 300ms fade-slide-up, staggered. And if a user has their OS set to reduced motion, all animations are suppressed. That's a WCAG accessibility requirement — handled automatically."

---

## SEGMENT 5 — The Code Quality Showcase (2 minutes)

*[Switch to VS Code — open `src/app/services/weather.service.ts`]*

> "Let me show you the generated code. This is the WeatherService — 163 lines."

*[Highlight the interface declarations]*
> "Starts with TypeScript interfaces. WeatherData, WeatherCache, GeocodingResult — all explicitly typed. No 'any' types."

*[Highlight the WMO_CODES constant]*
> "This is the WMO Weather Code lookup table — 24 entries mapping meteorological codes to human-readable descriptions and emoji icons. The AI researched the WMO standard and implemented it correctly."

*[Highlight the getCache() method]*
> "This is the caching logic. It reads from localStorage, parses JSON, checks if the data is less than 60 minutes old, and returns fresh data or null. All wrapped in a try/catch so a corrupted cache never crashes the app."

*[Open `src/styles.css`, scroll to skeleton animation]*
> "And here's the CSS — the skeleton shimmer animation. While weather data is loading, you see a pulsing grey shape the exact size of the widget. This is a proper loading UX pattern. Generated. Not copied from Stack Overflow — generated from the spec's requirement that said 'shimmering shape matching the widget's dimensions.'"

## SEGMENT 7 — The Offline Scenario (1 minute)
